1 GasCar elma project
2 ===

4 [Elma](http://klavinslab.org/elma) is an event loop manager designed to teach embedded and reactive systems programming. This repository contains an example Elma project, showing how to use many Elma features, as well as illustrating how to arrange the files and directories in an Elma project.
5 
6 The source code for this project is available [on github](https://github.com/klavinslab/elma_project).
7 
8 [GasCar](https://github.com/fyiyu091/elma_project) is a project that uses the Elma system to simulate the gascar model.

20 To run the gascar model, type

22  bin/gascar

24 The gascar is controlled via the keyboard, via these keys:
25 - **z**: start the gascar (to active state)
26 - **w**: move up
27 - **s**: move down
28 - **d**: move right
29 - **a**: move left
30 - **q**: quit
31 - **f**: to fillgas state
32 - **a**: add gas
33 - **r**: back to active state
34 - **p**: park to off state
35 
36 Architecture
37 ---
38 - The gascar project is essentially a statemachine that has three states: off, active and fillgas.
39 - Initially, the gascar model will starts at off state, event "start(press z)" will trigger the model to active state with 10 units of gases.
40 - In the active state, you can use "wsad" to move the car(simplified by "*"), note each movement will burn down one unit gas. Event "addgas(press f)" will trigger the model to fillgas state.
41 - In the fillgas state, you can press 'a' to add gas. Event "restart(press r)" will trigger the model back to active state.
42 - Besides, event "park(press p)" will trigger the model from active to off state.
43 - In any of the three states, you can press "q" to exit the gascar model simulation.
44 - Warning: Movement while the gas is at 0 level will prompt the error message on the screen and exit the model, so please remember to add gas!
